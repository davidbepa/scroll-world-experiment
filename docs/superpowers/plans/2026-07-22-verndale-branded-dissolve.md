# Verndale Branded Media and Complementary Dissolve Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Preserve authentic featured-client signage in the approved cinematic assets, align the reproducible generation contract with that decision, and replace the scroll engine's layer switch with a true complementary dissolve.

**Architecture:** Keep the generation policy centralized in `render/prompts.mjs`, with `src/cases.js` supplying case-specific subjects and `scripts/export-handoff.mjs` regenerating the standalone pack. Put the dissolve equation in a pure `segmentBlendWeights(position, segments, crossfade)` helper in `src/timeline.js`; `src/scroll-world.js` consumes those weights while assigning each media layer a stable index-based stacking order.

**Tech Stack:** Vanilla ECMAScript modules, Node.js built-in test runner, static HTML/CSS, FFmpeg asset verification, in-app browser QA.

## Global Constraints

- Work on the existing `main` branch; do not create a worktree or feature branch.
- Keep all six WebP posters and eleven H.264 clips byte-for-byte unchanged; no rerenders or image edits are in scope.
- This remains a desktop-film build with poster-led phone fallback; do not add native 9:16 or center-cropped mobile video.
- Featured-client identity may appear only when physically integrated into architectural signage, venue identity, or a product or vehicle marque for that case.
- Captions, typographic overlays, synthetic labels, promotional copy, watermarks, unrelated logos, and unrelated brand marks remain prohibited.
- Every seam uses a `0.08`-viewport-height band centered on the boundary. After smoothstep easing, incoming opacity is `t`, outgoing opacity is `1 - t`, and the two weights sum to `1`.
- Media stacking order must remain stable when the active segment changes.
- Preserve unrelated untracked files: `.agents/`, `render/ChatGPT.dmg`, and `skills-lock.json`.

- [ ] **Step 3: Add the pure complementary-weight calculation**

Append this function to `src/timeline.js` immediately after `smoothstep`:

```js
export function segmentBlendWeights(position, segments, crossfade) {
  const weights = Array(segments.length).fill(0);
  if (!segments.length) return weights;
  const band = Number.isFinite(crossfade) ? Math.max(0, crossfade) : 0;

  if (band > 0) {
    for (let index = 0; index < segments.length - 1; index += 1) {
      const boundary = segments[index + 1].start;
      const bandStart = boundary - band / 2;
      const bandEnd = boundary + band / 2;
      if (position < bandStart || position > bandEnd) continue;
      const incoming = smoothstep((position - bandStart) / band);
      weights[index] = 1 - incoming;
      weights[index + 1] = incoming;
      return weights;
    }
  }

  let active = 0;
  for (let index = 1; index < segments.length; index += 1) {
    if (position >= segments[index].start) active = index;
  }
  weights[active] = 1;
  return weights;
}
```
